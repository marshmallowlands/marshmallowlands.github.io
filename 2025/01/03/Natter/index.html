使用 Natter 脚本在无公网 IP 环境上实现端口映射

⚠ 注意
在开始操作前，您必须拥有对应的网络技能，并且您已经确认，您所在的网络环境必须是
全锥型的 NAT，否则您无法继续这个流程。此操作是强制性的。

前言

由于 IPv4 地址池耗尽的原因，现在，三大运营商一般情况下都不会给家庭用户公网 IP 了，这导致有很多需要公网 IP 用的服务，都用不了了。

一般情况下，除了和运营商再去要公网 IP，或者是使用内网穿透程序，基本上没有使用其他方法。

不过呢，现在又有新东西可以玩了，他呢，就是今天我们要说的脚本：Natter。

什么是 NAT（网络地址转换）？

NAT，是 Network Address Translation 的缩写，翻译为网络地址转换。是一种将私有网络地址转换为公有网络地址的技术，目的是为解决 IPv4 地址不足的问题。其解决方案是将多个设备处于一个局域网内，通过共享一个公有网络 IP 地址，以访问外部网络。

若要了解 NAT 的更多信息，请自行上网搜索 NAT 或是 网络地址转换。

Fullcone NAT ？这是啥玩意儿啊 ？

举个简单的例子，比如，你从小区门出来，但是呢，小区有闸机，不要在意，举个例子而已。
这个闸机呢，就是运营商的 NAT 设备了，但是呢这个闸机有等待时间，也就是说，有端口回收等待时间，
别人呢，就可以趁着闸机还开着，可以自由出入，有人通过，就会延长关闭时间。时间到了，就不让外人进入了。

想再次访问，你得重新从闸机出一下了，但是是从另一个闸机出，假设有 65535 个闸机，网络的端口号 65535 个可用，
你每次出闸机的出口都是不一样的。

这 Natter 程序是啥来头

Natter，是 Github 用户 MikeWang000000 编写的一个 Python 脚本程序，主要用途就是来在全锥型 NAT（Fullcone NAT）的网络环境下，通过一种 “打洞” 的方式，将本地网络的端口公开到至互联网上，比如公开 HTTP 网页服务器，Minecraft 我的世界服务器等服务。

想知道怎么用？那就接着往下看吧！

使用方法

在使用之前，你需要满足这些条件。

  • 全锥型 NAT、Fullcone NAT 的网络环境
  • 设备可以安装和运行 Python
  • 路由器或光猫设置 DMZ IP 到要使用 Natter 的设备。
  • 重中之重：按照本文的步骤，都正确无误设置完毕后，保持程序的运行。
